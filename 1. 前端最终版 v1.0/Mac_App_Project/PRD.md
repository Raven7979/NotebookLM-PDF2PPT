# PDFtoPPTX 1.2 产品需求文档 (PRD)

## 1. 简介

### 1.1 项目概览
PDFtoPPTX 是一款 macOS 应用程序，旨在将 PDF 文档转换为可编辑的 PowerPoint (PPTX) 演示文稿。与将页面视为静态图片或生成破碎文本框的标准转换器不同，本工具利用 AI 智能地将文字与背景分离。它通过 AI 修复（Inpainting）移除页面图片中的原始文字，并覆盖上内容、字号、颜色和位置相匹配的可编辑文本框，从而生成高质量、可编辑的幻灯片。

### 1.2 目标
- **高保真：** 保持原始 PDF 的视觉外观。
- **可编辑性：** 确保输出的 PPTX 中的文字内容完全可编辑。
- **纯净背景：** 利用 AI 移除背景图片中的原始文字，避免出现“重影”或双重文字。
- **自动化：** 简化转换流程，最大程度减少用户干预。

## 2. 用户流程

1.  **启动：** 用户打开 macOS 应用程序。
2.  **配置：** 默认使用 Nano banana Pro API 处理 AI 功能，确保极致的文字擦除效果。用户不再需要手动配置千问。
3.  **导入：** 用户选择要转换的 PDF 文件。
4.  **处理：**
    *   应用从 PDF 中提取页面。
    *   **阶段 1（分析）：** AI 和本地 OCR 分析文字内容、位置和样式。
    *   **阶段 2（修复/去字）：** AI 移除背景图片中的文字和水印。
    *   **阶段 3（生成）：** 应用构建 PPTX 文件。
5.  **反馈：** 应用显示实时进度（当前页码、预计剩余时间）。
6.  **导出：** 用户保存生成的 PPTX 文件。

## 3. 功能需求

### 3.1 PDF 处理
- **输入：** 支持标准 PDF 文件。
- **页面提取：** 将 PDF 页面转换为高分辨率图片以供处理。
- **限制：** 支持批量处理（当前限制为每任务 15 页，以确保稳定性）。

### 3.2 文字识别与分析（混合方案）
- **本地 OCR (Apple Vision)：**
    -   **文字位置 (Bounding Box)** 和 **旋转角度** 的主要来源。
    -   用于 **颜色采样**（文字颜色、背景颜色、描边颜色）。
    -   检测文字方向 (OBB - 定向边界框)。
- **AI 分析 (阿里云 Qwen-VL-Max)：**
    -   修正文字内容（纠正 OCR 错误）。
    -   检测语义样式（例如：识别粗体/标题）。
    -   与本地 OCR 数据合并，创建最准确的文字块表示。
- **分类：**
    -   自动检测 **水印**（例如角落里的 "notebook", "lm"），将其移除但不再重新生成为文字。
    -   自动检测 **Logo**，将其保留在背景中或单独处理。

### 3.3 图像修复（去字）
- **AI 修复 (阿里云 Wanx2.1)：**
    -   根据检测到的文字块生成遮罩（Mask）。
    -   将图片和遮罩发送到云端 API，以“擦除”文字并自然填充空缺。
- **降级机制：**
    -   如果 AI 失败或被禁用，回退到简单的本地算法（基于周围像素的颜色填充）。

### 3.4 PPTX 生成
- **幻灯片母版：** 创建干净的幻灯片布局。
- **背景：** 将“修复后的图片”（无文字版）设置为幻灯片背景。
- **文字重构：**
    -   生成原生 PowerPoint 文本框 (`<p:sp>`)。
    -   **映射属性：**
        -   内容（文本字符串）
        -   位置 (X, Y)
        -   尺寸 (宽, 高)
        -   字号（相对于幻灯片高度缩放）
        -   字体颜色（采样得到的十六进制值）
        -   旋转角度
        -   粗体状态
        -   描边/轮廓（如果检测到）
- **输出：** 生成有效的 `.pptx` 文件（压缩的 XML 结构）。

## 4. 技术架构

### 4.1 技术栈
- **操作系统：** macOS (12.0+)
- **语言：** Swift (UI 使用 SwiftUI)
- **核心框架：**
    -   `PDFKit`：PDF 处理。
    -   `Vision`：本地 OCR。
    -   `CoreImage`：图像处理。
    -   `AppKit`：系统集成。

### 4.2 核心模块
- **`ConversionViewModel`：** 协调整个流程，管理状态，处理错误报告。
- **`PDFProcessor`：** 处理 PDF 到图片的转换。
- **`OCREngine`：** 封装 Apple Vision 框架，用于文字检测和颜色分析。
- **`QwenService`：** 阿里云 API 接口层（Qwen-VL-Max 用于分析，Wanx2.1 用于图像修复）。
- **`InpaintingEngine`：** 本地去字降级方案。
- **`PPTXGenerator`：** 手动构建 PPTX 文件的 XML 结构（避免依赖第三方 PPT 库，以实现最大程度的控制）。

### 4.3 数据模型
- **`TextBlock`：** 存储归一化坐标、文本、置信度、颜色和样式信息。
- **`ProcessedPage`：** 保存原始图片、处理后（干净）的图片以及 TextBlock 列表。

## 5. 非功能性需求
- **性能：**
    -   所有耗时操作使用 Async/Await，保持 UI 响应。
    -   尽可能并行处理（目前为了管理 API 速率限制，每页处理仍为顺序执行）。
- **安全性：**
    -   API Key 存储在 `UserDefaults` 中。
    -   通过 HTTPS 请求阿里云 API。
    -   沙盒文件访问（Security Scoped Resources）。

## 6. 未来改进计划
- **并行处理：** 优化流水线以并发处理多个页面。
- **字体匹配：** 尝试匹配字体族（目前默认为 "Source Han Sans SC"）。
- **矢量图形：** 将 PDF 中的矢量形状转换为 PPTX 形状，而不是将其栅格化到背景中。
- **复杂布局：** 更好地处理表格和多栏布局。
